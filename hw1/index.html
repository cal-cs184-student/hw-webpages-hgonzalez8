Overview

In this homework, I implemented the razterization of triangles, both with and without supersampling, along with some transformations. I thought this project was something new and interesting that I got to work on. I learned how to implement supersampling and was able to see the visual changes that occur and how it changes the way the pixels are shown.

Task 1: Drawing Single-Color Triangles

Rasterizing triangles involves converting a shape into some pixels on the screen. There are multiple steps involved in the process. The steps I used started with creating a bounding box by getting the minimum and maximum of both the x and y coordinates to create a subsection of pixels to test instead of the entire frame buffer. This will make it more faster to process. By using the floor of the minimums and the ceiling of the maximums makes sure that no part of the triangle is cut off. I had to loop through the bounding box and sample the middle of the pixel. I had to check if the pixel was in the triangle based on the point in triangle test and fill it if it was.

Task 2: Antialiasing by Supersampling

Using the same method of finding the bounding box as task 1, I use the minimums and maximums of x and y. Since I was using supersampling, I had to use sub-pixels by finding the dimensions of the sib-pixel grid. There are sqrt(sample_rate) * sqrt(sample_rate) grid locations in the pixel so I used sqrt(sample_rate) as the dimension. In addition to looping through the bounding box, I had to also go through the sub-pixels. The middle of each subpixel can be found by using the index of the subpixel, the dimensions, and 0.5 to get the middle of the pixel. The point in triangle test was used and this time instead of using the fill pixel function, I used the sample buffer so that each subpixel color would be able to be averaged out to get combined into a color for the main pixel.
