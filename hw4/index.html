Overview

This homework involved building a cloth simulation system and rendering it realistically. I modeled the cloth as a grid of point masses connected by springs, which allowed it to move and deform based on physical forces like gravity and tension. I simulated its motion over time using numerical integration, where forces like gravity, spring forces, and damping determine how each point mass moves at every timestep. I also added collision handling so the cloth can interact with objects like spheres, planes, and even itself. Collision handling required detecting intersections and applying correction offsets so the cloth doesn't pass through surfaces or collapse into itself. To make self-collision efficient, spatial partitioning was used to avoid checking every pair of points. I also explored how different parameters, such as spring stiffness, density, and damping, affect the cloth's behavior and appearance. I implemented shaders to improve the visual appearance of the cloth. This included lighting models like Blinn-Phong for realistic shading, as well as techniques like texture mapping, bump mapping, and displacement mapping to enhance visual detail. Using these lighting models and texture techniques, the simulated cloth was able to look more detailed and realistic.

Part 2: Simulation via numerical integration

These are the differences in the cloth behavior that I observed when experimenting with some parameters in the simulation.
  1. Effects of changing ks:
    • High ks - There was less movement as the cloth fell as well as fewer, wider folds. The cloth resists deformation more strongly, since the spring forces are larger. It sticks out more around the sides and forms larger folds.
    • Low ks - There was more movement in the cloth, like swinging, and more folds which were also deeper. The springs are weaker, so the cloth deforms more easily. This leads to more motion, deeper folds, and a more wrinkled appearance.
  2. Effects of changing density:
    • Higher density - When the cloth was hanging, there was a deeper curve between the pinned points. This is because increasing the mass of each point increased the effects of gravity, making the cloth sag more.
    • Lower density - When going from high to low density, the cloth jumps up a little bit which made a shallower curve between the pinned points. This reduced the gravitational force, so the cloth appears lighter and less stretched.
  3. Effects of changing damping:
    • More damping - Increasing damping removes more energy from the system over time, which caused the cloth to fall more slowly and settle smoothly.
    • Less damping - Decreasing damping preserves more energy, so the cloth moves more quickly and oscillates more before coming to rest. The cloth fell a lot faster.

Part 3: Handling collisions with other objects

For handling collisions with spheres, I used the sphere's origin and radius. First, I compute the vector from the sphere's center to the point mass to determine its direction and distance from the center. I then check if a collision has occurred by comparing this distance to the radius.
  • if the distance is less than the radius, the point mass is inside the sphere
If a collision occurs, I compute a correction by normalizing this direction vector and scaling it so the point mass lies exactly on the surface of the sphere. This gives the corrected position. I then compute a correction vector from the previous position of the point mass to this new position. I apply this correction to the point mass's last position, scaled by a factor of \( 1 - f \), to account for friction.
For handling collisions with planes, I used the plane's point and normal vector. I compute the vector from a point on the plane to the point mass and take the dot product with the plane normal to determine the signed distance from the plane.
  • If this value is negative, the point mass is below the plane and a collision has occurred.
When a collision is detected, I compute a correction vector that moves the point mass above the plane by a small offset along the normal direction. I then compute the displacement from the last position to this corrected position and decompose it into normal and tangential components. The normal component ensures the point mass is above the surface, while the tangential component is scaled by friction to simulate realistic sliding along the plane. The final position is computed by adding the normal correction and the friction-scaled tangential component to the point mass's last position.

Part 4: Handling self-collisions

To handle self-collisions, I used the spatial map that we build to divide the space into 3D box volumes which contain the point masses that are located within that box. Using this map, I get the box containing the current point mass, then check each point mass in the same box, and make sure that it is not the same point mass we are comparing it to. I check the distance between two point masses to check that it is at least twice the thickness of the cloth. If it is not, I make a correction vector that moves the current point mass in the direction away from the other point mass by the amount that is needed for it to be at least twice the thickness away. This correction vector is added to a running total that sums together all the correction vectors from the point masses that are too close to current point mass. I also keep count of the number of correction vectors that are made since the average of the correction vectors scaled by the number of simulation steps is the amount and direction that the current point mass will move.

Part 5: Shaders

A shader program is run on the GPU to render computer graphics. It determines how objects appear by controlling lighting, color, and surface detail. Vertex shaders work with the location and geometry of vertices by applying transformations to determine the final position of each vertex in screen space. The attributes of the vertices are then passed as input to the fragment shader. The fragment shader uses these attributes to determine the final color of each pixel based on lighting and material properties. The Blinn-Phong shading model is a combination of ambient, diffuse, and specular shading to create a more realistic looking scene. The ambient component provides general illumination across the object, the diffuse component depends on the angle between the light and the surface normal, and the specular component creates highlights that simulate shininess. Together, these components determine the final color of the surface. For texture techniques, texture mapping applies an image to the surface to add color detail. Bump mapping modifies the surface normals to simulate small surface details without changing the actual geometry, while displacement mapping actually changes the positions of vertices, creating real geometric variation on the surface. Because of this, displacement mapping produces more realistic depth and silhouettes compared to bump mapping, especially at higher mesh resolutions. Based on what I can see, the difference between the different resolutions is that higher resolution meshes produce smoother and more detailed results, while lower resolution meshes appear coarser. I also noticed some Moiré patterns on the sphere at higher resolutions due to fine repeating details. The difference between bump mapping and displacement mapping is that displacement mapping creates actual geometric changes, while bump mapping only affects the lighting appearance.
