Part 3: Handling collisions with other objects

For handling collisions with spheres I did used components of the sphere that were available like the origin and radius to do the following:
  1. I found the tangent point - the surface intersection point
      2. I created a vector from the sphere origin to the point mass's position to get the direction of the possible extended path to the sphere's surface.
      I get the length of the vector to get the distance between the point mass's position and the sphere's origin.
  2. I checked if the point mass collided with the sphere
      3. To check, I compare the distance with the radius of the sphere : if the distance is less than the radius, the point mass is inside the sphere
  3. If they have collided, I make a correction vector : the distance needs to be more than the radius distance away from the origin, so I calculate how far the point mass is from the sphere's surface.
      4. Using the direction found before, we normalize it to get a unit vector, then scale it by the amount the point mass needs to move.
      This vector is added to the current position of the point mass to move it to where it doesn't collide with the sphere : this gives the new position that the point mass should be at
      I found the vector between the last position of the point mass and the new position to get the amount that the point mass should move from its last position : this is the correction vector
  4. Apply the correction vector to the point mass's last position and also scale it down by friction \( 1 - f \) : this updates the point mass's position at this timestep so it does not collide with the sphere

For handling collision with planes I used the components of the plane like the point and the normal to do the following:

  1. I found the direction and distance from the point on the plane to the current position of the point mass.
  2. I use the dot product of that vector and the normal of the plane to find the "vertical" distance from the plane that the point mass is at. The result is checked to see if the point mass collided with the plane.
    • if the result is positive: the point mass is above the plane.
    • if the result is negative: the point mass is below the plane -- the point mass collided with the plane
  3. If there was a collision, I make a correction vector: the total distance the point mass should travel to get above the plane would be the distance and the surface offset combined since we want the point mass to be a bit above the plane. We multiply this by the plane vector because we want to go in the direction that the plane is facing.
  4. The correction vector is added to the current point mass position to get it above the plane.
  5. A new vector is made from the last position of the point mass to the newly corrected position.
  6. Get the "vertical" and "horizontal components of that new vector.
    • To find the vertical component: get the dot product of that vector and the plane normal vector to get the length and multiply it by the plane normal to get the direction.
    • To find the horizontal component: subtract the vertical vector from the original vector.
  7. Get the final point mass position by adding the last position, the height vector, and add the horizontal vector scaled by friction.

Part 4: Handling self-collisions

To handle self-collisions, I used the spacial map that we build to divide the space into 3D box volumes which contain the point masses that are located within that box. Using this map, I get the box that the current point mass we are working with is in, then check each point mass in the same box, and make sure that it is not the same point mass we are comparing it to. I check the distance between two point masses to check that it is at least twice the thickness of the cloth. If it is not, I make a correction vector that moves the current point mass in the direction away from the other point mass by the amount that is needed for it to be at least twice the thickness away. This correction vector is added to a running total that sums together all the correction vectors from the point masses that are too close to current point mass. I also keep count of the number of correction vectors that are made since the average of the correction vectors scaled by the number of simulation steps is the amount and direction that the current point mass will move.

Part 5: Shaders

A shader program is run on the GPU to render computer graphics. It deals with transformations to the lighting, color, and shapes of the scene. Vertex shaders work with the location and geometry of vertices by applying transformations to determine the final position of the vertex. The attributes of the vertices are passed as input to the fragment shader. The fragment shader works with the attributes to determine the texture and how the light affects the color that the pixel should be. It outputs the final color that the pixel should be. The Blinn-Phong shading is a combination of ambient, diffuse, and specular shading to create a more realistic looking scene. It is able to simulate any highlights there may be based on the material and lighting, creates a general brightness to the scene, and is able to get the final color based on the surface color and lighting.
